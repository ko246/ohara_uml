# ユースケース名
顧客ログイン

# ユースケース概要
顧客が自身のIDとパスワードを入力し、システムにログインする。

# アクター
顧客

# 事前条件
- 顧客が会員登録済みであること

# 事後条件
- 顧客がシステムにログイン状態となる

# 基本フロー
1. 顧客はログイン画面を表示する
2. 顧客はIDとパスワードを入力する
3. システムは入力情報を認証する
4. 認証が成功した場合、顧客はログイン状態となる
5. システムはメニュー画面を表示する

# 代替フロー
- IDまたはパスワードが誤っている場合、エラーメッセージを表示し再入力を促す

# 関連情報
- 顧客ID、顧客パスワード
