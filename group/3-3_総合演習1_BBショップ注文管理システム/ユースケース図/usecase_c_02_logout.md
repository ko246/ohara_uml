# ユースケース名
顧客ログアウト

# ユースケース概要
顧客がWeb販売システムから安全にログアウトする。

# アクター
顧客

# 事前条件
- システムがログイン状態となっている
- メニュー画面が表示される

# 事後条件
- システムがログアウト状態となる
- メニュー画面が表示される

# 基本フロー
1. システムは、メニュー画面を表示する
2. 顧客は、「ログアウト」ボタンを押す
3. システムは、ログアウト処理を実行する  
    - ログアウト処理が失敗した場合は代替フロー①へ
4. システムは、メニュー画面に遷移する

# 代替フロー
① ログアウト処理が失敗した場合はエラーメッセージを表示し、基本フロー1に戻る
